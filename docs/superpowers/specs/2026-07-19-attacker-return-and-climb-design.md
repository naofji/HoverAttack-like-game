# アタッカー帰還AI + ホバー登坂 設計書

日付: 2026-07-19

## 課題

アタッカー(特にヘビー)がプレイ中にマップ下部へ溜まっていく。

原因は「下方向はフリーパス、上方向はほぼ不可能」というラチェット構造:

- 追跡中(`stop_and_shoot` / `chase_and_jump`)は崖をそのまま歩いて落ちる
- ジャンプは1〜2タイル分しか登れず、落下分を取り戻せない
- ヘビーは `movementType: 'stop_and_shoot'` でホバー未使用・ジャンプ力最弱(-5.0)のため、一度落ちると復帰不能

## ゴール

- 非戦闘時: スポーン地点(持ち場)周辺に帰還し、マップ全体の敵分布を維持する
- 戦闘時: ターゲットが上にいれば垂直追従できる
- 制約: ワープ・瞬間移動の演出は禁止。既存の物理(重力・ホバー燃料)の範囲で登る

## 設計

### 1. 持ち場の記憶

`EnemyAttacker` コンストラクタで `homeX / homeY` にスポーン座標を保存する。マップ・スポーン側の変更は不要。

### 2. AIステート 'return' の追加

判定優先順位: **chase**(視界内にターゲット) > **return** > **patrol**

- 帰還開始条件(いずれか):
  - `y > homeY + 6タイル(96px)` — 持ち場より6タイル以上「下」にいる
  - `|x - homeX| > 20タイル(320px)`
- 帰還完了条件: 持ち場から2タイル(32px)以内(垂直・水平とも)
- 開始6タイル / 完了2タイルのヒステリシスで境界での状態フリッカを防ぐ

上方向のズレは問題にならない(重力で自然に解消される)ため、垂直条件は下方向のみ。

### 3. ホバー登坂ロジック(共通ヘルパー)

`_climbToward(targetX, targetY)` を実装し、returnステートと chase中の垂直追従の両方から呼ぶ:

1. 地上: 燃料が40%未満なら待機(回復を待つ)。40%以上でジャンプして離陸
2. 空中: 目標高度より下にいる間 `hovering = true` にして `vy -= climbThrust`、燃料消費。上昇速度上限は既存準拠(-4.0)
3. 燃料切れ → 落下・着地 → 回復待ち → 再上昇。段階的に登る見た目になりワープ感はない
4. 水平: `targetX` へ `maxSpeed` で歩く/飛ぶ。壁・天井衝突時は既存の反転処理(`_moveAndCollide`)に任せる。経路探索(BFS)は使わない

既存のホバー描画(排気エフェクト)がそのまま流用される。

### 4. 追跡中の改善

- **崖降下の抑制**: chase中の崖チェック(`_moveAndCollide` の cliff check)で、ターゲットが自分より下にいる場合(`dy > 16px`)のみ崖から降りる。同高度・上にいる場合は反転する。これがラチェットの「下り側」を塞ぐ
- **垂直追従**: ターゲットが2タイル(32px)以上上にいて追いつけない場合、全 `movementType` で `_climbToward` を使う(現状ヘビーはジャンプのみで届かない)

### 5. タイプ別パラメータ

`ENEMY_ATTACKER_TYPES` の各typeに `climbThrust` を追加:

| type      | climbThrust | 意図 |
|-----------|------------|------|
| standard  | 0.55       | 標準 |
| heavy     | 0.45       | 登りが遅い(重量級の個性は維持しつつ復帰は可能に) |
| rival     | 0.65       | 機動力最高 |
| artillery | 0.5        | やや鈍重 |

既存の `chase_and_jump` / `skirmish` / `zigzag_chase` のホバー処理(固定 0.5〜0.6)も `climbThrust` に置き換えて一元化する。

### 6. テスト

モックの map / game で数百フレームのシミュレーションを回すユニットテスト(既存 `tests/` のスタイルに合わせる):

- 持ち場より8タイル下に置いたアタッカーが、規定フレーム内(例: 1800フレーム)に `homeY + 2タイル` 以内へ復帰する
- chase中、ターゲットが4タイル上にいるとき、アタッカーの高度が時間とともに上がる
- 帰還完了後は patrol に戻り、持ち場±6タイル内に留まり続ける
- chase中、ターゲットが同高度のとき崖から降りない(x が崖の先へ進まない)

## 変更ファイル

- `src/js/entities/EnemyAttacker.js` — homeX/homeY、returnステート、`_climbToward`、崖降下抑制、垂直追従
- `src/js/utils/Constants.js` — `climbThrust` 追加、帰還閾値の定数
- `tests/` — シミュレーションテスト追加

## スコープ外

- スポーン位置の再設計(現状のマップ定義のまま)
- 他の敵種(タンク・ドローン・タレット)の挙動変更
- BFS経路探索による移動(発射方向計算での既存利用はそのまま)
