# ホバーアタック風 HTMLゲーム 仕様書

## 1. 概要
- **ジャンル**: サイドビュー・ロボットアクションシューティングゲーム
- **プラットフォーム**: Webブラウザ (HTML5 Canvas / JavaScript ES6 Modules)
- **モデル**: 1984年 X1用ソフト「ホバーアタック」（I/Oバージョン）
- **目的**: プレイヤーはアタッカー（ロボット）とキャリアー（補給母艦）を交互に操作し、破壊可能な洞窟地形を切り開きながら右下方向の最深部に存在する**敵メインベース**を目指し、これを破壊する。破壊後に現れるフラッグを奪取したらクリア。
- **コアコンセプト**: 限られた弾薬とホバー燃料を管理しながら、キャリアーを安全に前進させ、アタッカーで偵察・戦闘を行う「二機一体の戦略アクション」。

### 2.2 HUD要素の詳細

| 要素 | 位置 | 表示内容 |
|---|---|---|
| **タイトル** | 上段左 | ゲームタイトル「HOVER ATTACK」 |
| **SCORE** | 上段右 | 現在のスコア（7桁表示） |
| **GRENADE** | 中段左 | グレネード残弾数（数字表示） |
| **MISSILE** | 中段中央 | ミサイル残弾数（数字表示） |
| **HOVER** | 中段右 | ホバー燃料残量（バーゲージ＋数値） |
| **TIME** | 下段左 | 経過タイム（MM:SS.FF形式） |
| **ATTACKER** | 下段中左 | アタッカー残機数 |
| **CARRIER** | 下段中右 | キャリアー残機数。横に耐久値（HP）を示す**カラーバー**を表示 |
| **MISSION** | 下段右 | 破壊したミッション目標数 |

- **キャリア方向指示器**: キャリアーが画面（カメラ）外にいる場合、画面の端にキャリアーの方向を示す**黄色の矢印**が表示される。

### 2.3 フォント・ビジュアルスタイル
- HUD文字はレトロなモノスペースフォント（Courier New系）で描画。
- ゲーム画面はピクセルアート調のレンダリング（`image-rendering: pixelated`）。
- 全体の背景色は暗黒（`#111`〜`#222`）で洞窟の暗さを表現。

## 3. 登場キャラクター・オブジェクト

### 3.1 アタッカー（プレイヤーロボット）
プレイヤーが直接操作する、高い機動力と攻撃力を持つメイン戦闘ユニット。

- **デザイン**: 白を基調とした小型スリムな二足歩行ロボット。肩にバズーカ砲（ミサイル/グレネード発射機構）、背中にホバー用バックパックを装備。完全な2Dサイドビュー。
  - ![Robot Concept Art v5](C:\Users\naoya\.gemini\antigravity\brain\c41c9956-c72a-494e-bb24-3ec6bb866b97\robot_concept_art_v5_1772381478644.png)
- **スプライトサイズ**: ゲーム画面上では約16×16〜24×24ピクセル程度の小さなドット絵として描画。コンセプトアートの約1/5スケール。
- **歩行・ポーズアニメーション**: スプライト画像切り替えではなく、プログラムで動的に足（線）を描画。
  - **歩行**: 4コマの交互歩行アニメーション。
  - **停止・着地**: 着地時には自動的に足を閉じた直立状態（休め姿勢）に復帰。
  - **しゃがみ・ドッキング**: キャリアーへのドッキング時、または高所からの落下（スタン）時に正面向きの「カエル足（ガニ股）」ポーズを取る。
- **ホバリング表現**: ホバー中はバックパックからシアン色の排気パーティクルが描画される。さらに、空中での移動速度（慣性）に応じて、プログラム制御により下肢が振り子のように揺れるリアルな挙動を再現。
- **パラメータ**:

| パラメータ | 初期値 | 説明 |
|---|---|---|
| ミサイル残弾 | 24 | キャリアーとのドッキングで補給 |
| グレネード残弾 | 12 | キャリアーとのドッキングで補給 |
| ホバー燃料 | 100% | 使用中は減少、地上で自然回復なし。キャリアーとのドッキングで全回復 |
| 移動速度 | — | 歩行時に慣性（滑り）を伴う |
| 耐久値(HP) | — | 被ダメージで減少し、被弾時は火花（Sparks）が散る。0になると撃破（残機-1）。残機0でゲームオーバー。 |

### 3.2 キャリアー（補給母艦）
アタッカーとドッキング可能な移動・補給ベース。ゲームの生命線。

- **デザイン**: 赤と青を基調とした大型飛行空母。上部にアタッカーを載せるための平らなプラットフォーム（荷台）を備える。完全な真横からの視点。

- **スプライトサイズ**: アタッカーより大幅に大きく、画面上で存在感のあるサイズ（約64×32ピクセル程度）。
- **パラメータ**:

| パラメータ | 説明 |
|---|---|
| 耐久値(HP) | 被ダメージで減少。0になるとキャリアー撃破（残機-1）。**キャリアーの全損＝ゲームオーバー** |
| 移動速度 | アタッカーの徒歩と同等かやや遅い |

#### 地上ユニット

|---|---|---|---|---|
| **小型戦車** | 黄/青 | 地面を左右に移動 | 直線弾（進行方向の180度内のみ） | 最も一般的な地上敵。地形に沿って移動する。時折砲身を上下に向ける。背後の敵には砲撃しない。**1面から出現する。** |
| **アタッカー（防禦型）** | 緑 | プレイヤー追跡（接近で停止） | ミサイル（精度低） | 装甲が厚いが動きが遅く、ジャンプ力が低い。プレイヤー付近に大まかに狙いをつける。**3面から出現する。** |
| **アタッカー（汎用型）** | 水色 | 常に歩行（端でジャンプ） | ミサイル（精度中） | 機動性はプレイヤーの半分程度。装甲が薄い。命中率は約50%。出現頻度が高い。**2面から出現する。** |

| **ドローン (クアッドコプター)** | 銀/赤 | ホバリング・不規則な高速ダッシュ | 弾/グレネード投下 | 空中を機敏にフワフワと飛行。プレイヤーと一定距離を保ちつつ、攻撃直前に一定時間静止して弾を撃つ。時折上空からグレネードも投下する。装甲は弱く容易に破壊可能。**4面から出現する。** |

#### 敵弾
- 小さな円形の弾（青または黄色）。
- 直線的または放物線的な軌道で飛来。

### 3.4 地形・環境

#### ブロックタイプ
| タイプ | 色 | 破壊可否 | 説明 |
|---|---|---|---|
| **通常ブロック** | 赤茶色 | ◯ 破壊可 | 洞窟を構成するメインブロック。ミサイル/グレネードで破壊可能 |
| **強化ブロック** | 青/シアン | △ 高耐久 | 特殊な構造物。破壊に複数発の攻撃が必要 |
| **破壊不能壁** | 灰色 | ✕ 破壊不可 | ステージの外周や特定のルート制限用。絶対に破壊できない |

#### 地形の特徴
- 洞窟は上下左右に入り組んだ構造で、全体として**右下方向へ進行**するレベルデザイン。
- プレイヤーはブロックを破壊して自らルートを切り開く必要がある。
- 天井・床はステージ外周として厚い破壊不能壁で覆われる。

### 3.5 トラップ（地雷・障害物）
地形上に設置された固定式の障害物。

| タイプ | 外見 | 効果 |
|---|---|---|
| **地雷** | 小型の点滅するオブジェクト | 接触またはミサイル等の爆発で起爆。周囲（約3.5マス）に**AoEダメージ**を与え、周囲の敵も巻き込む。他の地雷を巻き込んで連鎖爆発（誘爆）も起こす。宙に浮いている機体（キャリア、小型ホバー戦車）は接触判定を持たない。 |
| **電撃トラップ** | 周期的に発光するバリア | 一定間隔で通電し、触れるとダメージ（未実装） |

### 3.6 敵メインベース（最終目標）
- マップの最深部（主に右端エリア）に設置された巨大な敵の中枢拠点（アタッカーの約2倍の大きさ）。
- 中央に光り輝くコアを持ち、その外側を3重の防御壁（シールド）が覆っている構造。
- **防衛システム（ベースレーザー）**:
  - プレイヤーまたはキャリアが一定距離（約400px）まで近づくと、コアの中心にエネルギーを充填するモーション（光の粒子の収束）を開始します。
  - 約1.5秒（90フレーム）のチャージ後、ターゲットに向けてエメラルドグリーンの高速レーザーを放ちます。
  - レーザーは地形（ブロック）を無視して直進し、命中すると大ダメージ（HP50%相当）を与えます。
  - 発射後は一定時間のクールダウンが必要です。
- ミサイルやグレネードによるダメージは「一番外側の防御壁」から1枚ずつ剥がすように順番に減少していく。
- 3枚の防御壁をすべて破壊した後にコア本体へ攻撃を当てる（計4回のダメージ判定）ことでベースが破壊される。
- メインベースを破壊すると、その場所に**クリアフラッグ**が出現します。
- 出現したフラッグをアタッカーで奪取することで「MISSION COMPLETE」となりステージクリア。
- スコアとクリア数を引き継いだまま、Wキーまたはクリックで次の自動生成ミッションへ進行する。

## 4. 操作仕様 (PC向けキーボード・マウス操作)

本作はオリジナルの「3方向射撃（上・直進・下）」を発展させ、マウスによる直感的な360度エイミングシステムを採用した操作系とする。

### 4.1 キーボード操作（移動・アクション）

| キー | 通常時 | ドッキング時 |
|---|---|---|
| **A / ←** | アタッカーを左に移動 | キャリアーごと左に移動 |
| **D / →** | アタッカーを右に移動 | キャリアーごと右に移動 |
| **W** | バースト（接地時・燃料80%以上） | ドッキング解除（アタッカー発進）。※頭上に障害物がある場合はキャンセルされる |
| **Shift** | ホバー（空中で長押し、燃料消費） | — |
| **s** | ドッキング開始※ / 長押しでホバー燃料の回復速度UP（裏技） | — |
| **Space** | ミサイル発射 | — |
| **M** | 洞窟全体のオーバーレイ・ミニマップ表示（トグル式） | 同左 |

> ※ドッキング条件: アタッカーがキャリアーのプラットフォーム中央付近に接地していること。

#### 移動の物理挙動
- **地上歩行**: 地上ではドリフト（滑り）なし。キーを離すと即座に停止する。
- **空中制御**: 空中ではわずかな慣性（エアフリクション）が働く。
- **速さ制限**: 落下速度（+8.0）に上限がある。また、ホバー中の上昇速度（-6.0）にも上限リミッターがかかる。
- **重力**: 空中では常に下方向の重力が適用される。
- **バースト**: 燃料が80%以上ある場合のみ、接地中にWキーで大量の燃料を消費し瞬時に爆発的な加速（初速度）を得て跳躍する。使用後約0.5秒間はホバーが使用不可になる。
- **ホバー**: バースト後、約0.5秒経過してからWキーを長押しし続けるとホバーが起動し燃料を消費しながら上昇する。
  - **動的推力**: ホバーの推力は燃料残量に応じて変化する。満タン時は重力より強く（上昇可能）、残量が減ると次第に推力が低下し、最低-0.15まで弱まる。
- **しゃがみとスタン**: 一定以上の高さから落下して着地すると「ハードランディング」となり、一定時間しゃがみ状態（スタン）になる。しゃがみ・ドッキング中はバズーカ（ミサイル・グレネード）の発射ができない。
- **離脱時安全確認**: キャリアーから離れる（発進する）際、アタッカーが立ち上がるための頭上スペースにしきい値以上の障害物（ブロック）がある場合は、危険と判断して離脱アクションがキャンセルされる。

### 4.2 マウス操作（攻撃・向き）

| 操作 | 機能 |
|---|---|
| **カーソル移動** | 照準（クロスヘア）の操作 |
| **左クリック / Space** | 照準方向にミサイル発射 |
| **右クリック** | 照準方向にグレネード発射（放物線） |
| **向き自動制御** | 照準がアタッカーの左→左向き、右→右向きに自動切替 |

### 4.3 照準（クロスヘア）の仕様
- 赤色の半透明十字線で描画。
- ゲーム画面上に常時表示。
- マウスカーソルの位置にそのまま追従。
- **ロック解除**: クロスヘアがロック状態（右クリックなど）であっても、マウスを動かすことで即座にロックが解除され、自由なエイミングに戻る。

## 5. 武器・攻撃システム

### 5.1 ミサイル（メインウェポン）
| 項目 | 仕様 |
|---|---|
| 発射方法 | 左クリック |
| 軌道 | アタッカー位置から照準方向への**直線弾** |
| 速度 | 高速 |
| 威力 | 通常ブロック1発破壊。敵ユニットに中ダメージ |
| 弾数 | 有限（初期24発）。キャリアードッキングで全回復 |
| 同時発射数 | 画面内に最大2〜3発まで同時存在可 |

### 5.2 グレネード（サブウェポン）
| 項目 | 仕様 |
|---|---|
| 発射方法 | 右クリック |
| 軌道 | 照準方向への**放物線（アーク）弾**。投下後、地面や壁を跳ね返りながら転がる。 |
| 速度 | ミサイルより遅い |
| 威力 | 高い。広範囲に爆破。強化ブロックも破壊可能。爆風は範囲内の敵すべてに大ダメージを与えるが、自機やキャリアも巻き込まれるため注意が必要。 |
| 弾数 | 有限（初期12発）。キャリアードッキングで全回復 |
| 爆発範囲 | 着弾点を中心に周囲ブロックも巻き込み破壊。約2秒後に自動爆発。 |

### 5.3 キャリアー武装（自動防衛）
- 短射程の自動ミサイルを前方に発射（弾数無制限）。
- 射程は限定的で、接近する敵への自衛用。

### 5.4 爆発・ダメージエフェクト
- 着弾・破壊時に黄色〜白のピクセルクラスタが一瞬フリッカーするレトロ風爆発エフェクト。
- ブロック破壊時は破片パーティクル（小さな四角形）が飛散。
- 自機や敵機がダメージを受けた際、上方向にオレンジ色の火花（Sparks）が散る演出が発生。

## 6. 主なゲームメカニクス

### 6.1 ドッキングと補給システム
ゲームの根幹をなす最重要メカニクス。

```
【ドッキングシーケンス】

  アタッカーがキャリアー上部に接地
          ↓
     [S]キーでドッキング開始
          ↓
  ドッキング完了アニメーション
          ↓
  ミサイル / グレネード / ホバー燃料 全回復
          ↓
  [A][D]でキャリアーごと移動可（前線構築）
  ※この状態では射撃不可（しゃがみポーズ）
          ↓
  [W]キーでアタッカー発進（頭上クリア確認後）
```
### ドッキングとキャリア操作（W/A/S/Dキー）
プレイヤーは「キャリア」と呼ばれる補給母艦から出撃し、作戦を行います。

- **ドッキング（Sキー）**: キャリアの上に乗り、Sキーを押すことでドッキング状態になります。
  - ドッキング中は自動的にHP、ミサイル、手榴弾、ホバー燃料が満タンに回復します。
  - キャリアに乗ってA・Dキーで左右に安全に移動・横断が可能です。
- **アンドック（Wキー）**: ドッキング中にWキーを押すと、上空に飛び降りて単独行動を開始します。
- **キャリア防衛と敵の標的**:
  - 敵（戦車・アタッカー等）は距離に応じてプレイヤーまたはキャリアを標的として狙ってきます。
  - プレイヤーがドッキングしている間は、敵の攻撃はキャリアに集中しやすくなります。
- **キャリア破壊・強制離脱とリスポーン**:
  - 敵の攻撃によりキャリアのHPが0になるとキャリアは爆発し、残機（LIVES）が1減少します。
  - ドッキング中にキャリアが破壊された場合、プレイヤーは自動的に上空へ放り出され（強制離脱）、単独操作に戻ります。
  - キャリアは残機がある限り、初期のスタート地点付近にリスポーン（復活）します。キャリアの残機が0になった状態で破壊されると、プレイヤーの残機に関わらずゲームオーバーになります。掃討 → 安全確認 → キャリアーを前進 → 再ドッキングして補給、を繰り返す。

### 6.2 物理挙動とエンティティの相互干渉
- **重力**: アタッカーには常に下方向の重力が適用される。
- **ホバー**: 燃料を消費して重力に逆らい、上昇・滞空が可能。
- **慣性**: 移動停止後も少しだけ滑空する（空気抵抗/摩擦のシミュレーション）。
- **エンティティの相互干渉（当たり判定）**:
  - プレイヤーと敵キャラクター（アタッカー、戦車）はお互いにすり抜けない「剛体（ソリッド）」として干渉します。
  - 横からぶつかった場合は「壁」として機能し、押し返されます。
  - 敵の頭上にジャンプ・落下した場合は「足場」として着地することができ、敵の上に乗ったまま移動することが可能です。
  - 敵同士も同様に干渉し、例えばパトロール中の死角で敵同士が重なることを防ぎます（ぶつかると方向転換します）。さらに生成時（スポーン時）も重ならないようばらけて配置されます。
  - **フレンドリーファイア無効**: 敵味方の識別があり、敵が放った弾や爆風は他の敵ユニットにはダメージを与えません。
  - 戦車（EnemyTank）は1段差の壁であればスムーズに登り、1段差の崖であれば降りることができます（2段以上の崖は反転して引き返します）。
- **ホバー燃料の自動回復**: ホバーを使用していない間、燃料は自動的に回復する（回復速度: 0.5/frame、満タンまで約3.3秒）。キャリアーとのドッキング時は即座に全回復。
  - **🕹️ 裏技**: キャリアーから離れた状態で`S`キーを長押しすると、ホバー燃料の回復速度が約20%上昇する（0.6/frame）。わずかな差だが空中での滞空時間をほんの少し延長可能。キャリアー付近では通常のドッキング操作が優先される。
- **慣性**: 地上移動時、入力停止後も僅かに滑る（氷上ほどではない微慣性）。
- **しゃがみジャンプ**（オリジナル準拠）: 特定操作で通常より高いジャンプが可能（実装検討）。

### 6.3 破壊可能地形
洞窟を構成するブロックの多くは攻撃で破壊可能。

- **戦略的用途**:
  - ルート開拓: 壁を壊して新たな通路を作る。
  - 射線確保: 安全な位置から壁を壊し、敵を狙い撃つ。
  - 落とし穴: 足元のブロックを壊して下層へ降りる。
  - キャリアー通路確保: キャリアーが通れる幅の通路を確保する。
- **破壊演出**: ブロック破壊時は小さな破片パーティクルが散ってから消滅。

### 6.4 マウスエイムによる360度攻撃
- マウスカーソル位置 = 照準位置。アタッカーの位置から照準方向にミサイル/グレネードを発射。
- **移動と攻撃の独立制御**: 左に移動しながら右にミサイルを発射する「引き撃ち」、上昇しながら下方にグレネードを投げるなどの高度な戦術が可能。
- アタッカーの向き（左向き/右向き）は照準位置により自動制御される。

### 6.5 リソース管理
本ゲームはリソース管理がゲーム性の核。

| リソース | 消費条件 | 回復条件 | 枯渇時 |
|---|---|---|---|
| ミサイル | 左クリックで1発消費 | キャリアーとドッキング | 発射不可 |
| グレネード | 右クリックで1発消費 | キャリアーとドッキング | 発射不可 |
| ホバー燃料 | Shift長押しで継続消費 | **自動回復**（未使用時0.5/f）/ ドッキングで全回復 / S長押しで+20%(裏技) | ホバー不可、落下 |
| アタッカーHP | 被弾・トラップ接触で減少 | キャリアーとドッキング（※検討） | アタッカー撃破 |

### 6.6 スクロールとカメラ
- **追従カメラ**: アタッカー（操作対象キャラ）を中心にスムーズに追従。
- **補間**: 急なカメラジャンプを防ぐ線形補間（lerp）によるスムーズパン。
- **マップ境界クランプ**: カメラはマップ外には出ない。

## 7. ゲーム進行・フロー

### 7.1 ゲームフロー全体図
```
【タイトル画面】
      ↓ スタート
【ステージ開始】
  アタッカーはキャリアー上にドッキング状態で開始
      ↓
【メインゲームプレイ】
  ドッキング解除 → 探索・戦闘 → 補給 → 前進 を繰り返す
      ↓
  ┌─ アタッカー撃破 → 残機あり → キャリアーから再出撃
  │                 → 残機なし → キャリアーのみで継続（※検討）
  │
  ├─ キャリアー撃破 → 残機あり → キャリアー再出現
  │                 → 残機なし → ゲームオーバー
  │
  └─ 敵メインベース破壊 → ステージクリア
      ↓
【ステージクリア演出】
  "MISSION COMPLETE" と表示
      ↓
  [W]キー または [左クリック] で次のミッション（新しいマップ）へ進行
  ※スコア・ミッション数はそのまま引き継がれる
      ↓
【ネームエントリー（ハイスコア時）】
  "CONGRATULATIONS!" → 名前入力 → ハイスコアテーブル表示
      ↓
【タイトル画面 or 次ステージへ】
```

### 7.2 初期状態
- アタッカー残機: 3機
- キャリアー残機: 3機
- ドッキング状態でステージ左上付近からスタート。
- ミサイル・グレネード・ホバー燃料は満タン。

### 7.3 撃破・リスポーン
- **アタッカー撃破時**: 爆発エフェクト → 残機-1 → キャリアー上に新アタッカーが出現（ドッキング状態）。
  - **再出撃の即時無敵**: リスポーンした瞬間から一定時間（約2秒間）、**無敵状態となり機体が点滅**する。
  - **状態の完全リセット**: リスポーン時にはそれまでのホバリング状態、しゃがみ、スタン、および動作音（ホバー音等）がすべてクリアされ、クリーンな状態で再開される。
- **キャリアー撃破時**: 大爆発エフェクト → 残機-1 → （ドッキング中であれば同時にアタッカーも失う）。
- **ゲームオーバー条件**: キャリアーの残機が0の状態でキャリアーが撃破される。

### 7.4 スコアリング

| 行動 | スコア |
|---|---|
| 通常敵撃破 | 100〜500点（種類別） |
| 固定砲台撃破 | 300点 |
| 飛行ユニット撃破 | 500点 |
| ミッション目標破壊 | 1000点 |
| 敵メインベース破壊 | 5000点 |
| タイムボーナス | クリア時間に応じた加算 |

## 8. マップ・レベルデザイン

### 8.1 マップ仕様
- **タイル（グリッド）ベース**: 固定サイズのタイルで構成（例: 32×32px/タイル）。
- **マップ全体サイズ**: 横100タイル × 縦40タイル程度（3200×1280px）のスクロールマップ。
- **進行方向**: 全体として左上（スタート）→ 右下（ゴール）へ進む構成。
- **レイヤー構造**: 背景レイヤー（暗黒の洞窟背景）+ タイルレイヤー（ブロック）+ エンティティレイヤー（キャラクター、敵、トラップ）。

### 8.2 マップ生成
- **Rooms and Tunnels方式（アリの巣状構造・大空間特化）**を使用した手続き型生成アルゴリズムによるハイブリッド方式。
- ステージ構成手順:
  1. スタート（左上）、ゴール（右下）、敵配置用の巨大な大部屋を複数確定で配置。
  2. さらにランダムな位置に、非常に巨大な部屋（チャンバー）を35〜49個程度大量に生成。
  3. すべての部屋が孤立しないよう、太いトンネル（幅6〜11マス）で結びつけ、マップ全体の多くを「空洞」にする。
  4. セルオートマトンによるスムージングを数パス行い、直線的な壁を有機的で自然な洞窟の形状（アリの巣風）に削り出す。
  5. 空間が広すぎて足場がない場所に対して、ランダムに横長の**浮遊ブロック（プラットフォーム）**を多数配置する。
- その後、破壊可能なハードブロック、トラップ、敵ユニットを空洞やプラットフォーム上に配置する。
- 全体として、ステージには以下のエリア構成要素を含む:
  - **スタートエリア**: 左上に配置された超巨大な安全地帯。キャリアーの初期位置。
  - **中間通路/大空洞**: 太い通路と巨大な部屋が連なり、足場（プラットフォーム）が浮かぶ広大なメインエリア。
  - **敵配置エリア**: 敵ユニットとトラップが密集するランダム配置の巨大な危険地帯。
  - **ボスエリア**: 敵メインベースが設置された右下最深部の超巨大な空間。

### 8.3 全体マップ表示
- `M`キーで画面中央に広域マップを半透明のオーバーレイ表示。一度生成したオフスクリーンキャンバスを用いて軽量に描画。
- 地形の状態に加え、現在のアタッカー位置（白）、キャリアー位置（青）、生きている敵ユニットの位置（赤）がリアルタイムに表示される。

## 9. 技術仕様

### 9.1 描画
- HTML5 Canvas 2Dコンテキストによるレンダリング。
- `requestAnimationFrame`によるゲームループ（可変フレームレート、deltaTime制御）。
- 描画最適化: カメラ範囲内（ビューポート内）のタイルとエンティティのみ描画。

### 9.2 ゲームループ
```
requestAnimationFrame(loop)
  └── loop(timestamp)
       ├── deltaTime = timestamp - lastTime
       ├── update(deltaTime)
       │    ├── Input処理
       │    ├── 物理演算（重力、慣性、衝突判定）
       │    ├── エンティティ更新（プレイヤー、敵、弾）
       │    └── カメラ更新
       └── draw()
            ├── 背景描画
            ├── マップ描画（ビューポート範囲のみ）
            ├── エンティティ描画
            └── HUD描画（カメラ座標とは独立）
```

### 9.3 衝突判定
- タイルベースの矩形衝突判定（AABB）。
- エンティティ同士の衝突判定（弾 vs 敵、弾 vs ブロック、プレイヤー vs トラップ etc.）。

## 10. サウンド・SE（実装済み）

| イベント | SE概要 |
