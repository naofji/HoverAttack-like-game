# ホバーアタック風 HTMLゲーム 仕様書

## 1. 概要

- **ジャンル**: サイドビュー・ロボットアクションシューティングゲーム
- **プラットフォーム**: Webブラウザ (HTML5 Canvas / JavaScript ES6 Modules)
- **モデル**: 1984年 X1用ソフト「ホバーアタック」（I/Oバージョン）
- **目的**: プレイヤーはアタッカー（ロボット）とキャリアー（補給母艦）を交互に操作し、破壊可能な洞窟地形を切り開きながら右下方向の最深部に存在する**敵メインベース**を目指し、これを破壊する。破壊後に現れるフラッグを奪取したらクリア。
- **コアコンセプト**: 限られた弾薬とホバー燃料を管理しながら、キャリアーを安全に前進させ、アタッカーで偵察・戦闘を行う「二機一体の戦略アクション」。

### 2.2 HUD要素の詳細

| 要素 | 位置 | 表示内容 |
|---|---|---|
| タイトル | 1段目左 | ゲームタイトル「HOVER ATTACK」 |
| TIME | 1段目中左 | 経過タイム（MM:SS.FF形式） |
| MISSION | 1段目中右 | 破壊したミッション目標数 |
| SCORE | 1段目右 | 現在のスコア（7桁表示） |
| GRENADE | 2段目左 | グレネード残弾数（数字表示） |
| MISSILE / M-GUN | 2段目中左 | ミサイル残弾数とマシンガンの状況を表示。**Fキー**で手動切り替えが可能。切り替え時は現在選択されている武装がハイライト表示される。マシンガンへの切替時は自動的にリロードが開始される。 |
| A-AIM | MISSILE/M-GUN の直下（小） | **オートエイムユニット**有効中のみ表示。オレンジの細バーグラフで残り時間を表す。 |
| HOVER | 2段目中央 | ホバー燃料残量。右に行くほど太くなる三角形ゲージ（/\|）で表示。80%以上（バースト可能）になると色の境界線が白く発光（光彩）する。 |
| ATTACKER | 2段目中右 | アタッカー残機数。横に耐久値（HP）を緑（残りライフ）と赤（ダメージ）のカラーバーで表示。またフィールド上でも被弾時にポップアップ表示される。 |
| CARRIER | 2段目右 | キャリアー残機数。横に耐久値（HP）を緑（残りライフ）と赤（ダメージ）のカラーバーで表示。またフィールド上でも被弾時にポップアップ表示される。 |
| リペアキットアイコン | CARRIER の直下（小） | 所持中のキャリアリペアキット数を、緑色の小さなアイコン（7×7px・白十字付き）で1個ずつ並べて表示。所持ゼロ時は非表示。 |
| MAP | (Rキー) | **Rキー**で画面中央にマップを表示。プレイヤーが移動（A/D/Wキー）を開始すると自動的にフェードアウトしながら閉じる。 |

- **キャリア方向指示器**: キャリアーが画面（カメラ）外にいる場合、画面の端にキャリアーの方向を示す**黄色の矢印**が表示される。

### 2.3 フォント・ビジュアルスタイル

- HUD文字はレトロなモノスペースフォント（Courier New系）で描画。
- ゲーム画面はピクセルアート調のレンダリング（`image-rendering: pixelated`）。
- 全体の背景色は暗黒（`#111`〜`#222`）で洞窟の暗さを表現。

## 3. 登場キャラクター・オブジェクト

### 3.1 アタッカー（プレイヤーロボット）

プレイヤーが直接操作する、高い機動力と攻撃力を持つメイン戦闘ユニット。

- **デザイン**: 白を基調とした小型スリムな二足歩行ロボット。肩にバズーカ砲（ミサイル/グレネード発射機構）、背中にホバー用バックパックを装備。完全な2Dサイドビュー。
  - ![Robot Concept Art v5](C:\Users\naoya\.gemini\antigravity\brain\c41c9956-c72a-494e-bb24-3ec6bb866b97\robot_concept_art_v5_1772381478644.png)
- **スプライトサイズ**: ゲーム画面上では約16×16〜24×24ピクセル程度の小さなドット絵として描画。コンセプトアートの約1/5スケール。
- **歩行・ポーズアニメーション**: スプライト画像切り替えではなく、プログラムで動的に足（線）を描画。
  - **歩行**: 4コマの交互歩行アニメーション。
  - **停止・着地**: 着地時には自動的に足を閉じた直立状態（休め姿勢）に復帰。
  - **しゃがみ・ドッキング**: キャリアーへのドッキング時、または高所からの落下（スタン）時に正面向きの「カエル足（ガニ股）」ポーズを取る。
  - **空中（ジャンプ・ホバー時）**: 空中では移動速度（vx）の反対方向に足が慣性で流れる。前方に飛べば足は後方に、後方に飛べば足は前方に振られる。
- **ホバリング表現**: ホバー中は**バックパック下部の橙色ノズル**からシアン色の排気パーティクルが描画される（向きに応じてノズル位置が左右に切り替わる）。さらに、空中での移動速度（慣性）に応じて、プログラム制御により下肢が振り子のように揺れるリアルな挙動を再現。
- **パラメータ**:

| パラメータ | 初期値 | 説明 |
|---|---|---|
| ミサイル残弾 | 24 | キャリアーとのドッキングで補給 |
| グレネード残弾 | 12 | キャリアーとのドッキングで補給 |
| ホバー燃料 | 100% | 使用中は減少、地上で自然回復なし。キャリアーとのドッキングで全回復 |
| 移動速度 | — | 歩行時に慣性（滑り）を伴う |
| 耐久値(HP) | — | 被ダメージで減少し、被弾時は火花（Sparks）が散る。0になると撃破（残機-1）。残機0でゲームオーバー。 |

### 3.2 キャリアー（補給母艦）

アタッカーとドッキング可能な移動・補給ベース。ゲームの生命線。

- **デザイン**: 赤と青を基調とした大型飛行空母。上部にアタッカーを載せるための平らなプラットフォーム（荷台）を備える。完全な真横からの視点。
  - ![Carrier Concept Art v3](C:\Users\naoya\.gemini\antigravity\brain\c41c9956-c72a-494e-bb24-3ec6bb866b97\carrier_concept_art_v3_1772381226326.png)
- **スプライトサイズ**: アタッカーより大幅に大きく、画面上で存在感のあるサイズ（約64×32ピクセル程度）。
- **パラメータ**:

| パラメータ | 説明 |
|---|---|
| 耐久値(HP) | 被ダメージで減少。0になるとキャリアー撃破（残機-1）。**キャリアーの全損＝ゲームオーバー** |
| 移動速度 | アタッカーの徒歩と同等かやや遅い |
| 武装 | 短射程ミサイル（無制限）。自動防衛として前方に発射 |

> **重要**: キャリアーの残機が0の状態で撃破されるとゲームオーバーとなる。アタッカーの残機が残っていてもゲーム続行不可。キャリアーの保護は最優先事項。

### 3.4 地形・環境

#### ブロックタイプ

| タイプ | 色 | 破壊可否 | 説明 |
|---|---|---|---|
| **通常ブロック** | 赤茶色 | ◯ 破壊可 | 洞窟を構成するメインブロック。ミサイル/グレネードで破壊可能 |
| **強化ブロック** | 青/シアン | △ 高耐久（HP3） | 特殊な構造物。破壊に複数発の攻撃が必要。被弾するたびにクラックが2本ずつ増え（最大4本）、ダメージ状態を視覚的に表現する |
| **破壊不能壁** | 灰/光沢 | ✕ 破壊不可 | ステージの外周や特定のルート制限用。絶対に破壊できない |

#### ブロック描画

- **通常・強化ブロック（岩肌表現）**: 隣接する4方向のブロック有無に応じた**ネイバーアウェア描画**を採用。隣が空洞の辺のみ凹凸（ジャグドエッジ）を描画し、隣接空洞の角には面取り（チャンファー）を施して有機的な洞窟の岩肌を再現する。ブロックごとに決定論的なシードRNGで色のばらつき・クラック・エッジのジッターを生成するため、フレーム間で安定した見た目を保つ。また各ブロック内部に暗い三角形（2枚・必ず描画）と明るい三角形（1〜2枚・確率で描画）を重ね合わせ、岩石のゴツゴツとした立体的なファセット感（面の光と影）を表現する。その上にひび線が重なる。
- **破壊不能壁（光沢タイル表現）**: ランダム性を排除した人工的な光沢タイルとして描画。ベベル（面取り線）、内側のインセットライン、左上の鏡面ハイライトを4辺すべてに均一に描画し、人造ブロックであることを明示する。

#### 地形の特徴

- 洞窟は上下左右に入り組んだ構造で、全体として**右下方向へ進行**するレベルデザイン。
- プレイヤーはブロックを破壊して自らルートを切り開く必要がある。
- 天井・床はステージ外周として厚い破壊不能壁で覆われる。

### 3.5 アイテム

#### キャリアリペアキット

- **外見**: 16×16px（ブロックと同サイズ）の角丸四角形。緑色（明度がパルスして光り輝くグロー付き）。中央に白い十字マーク（修理シンボル）。
- **ドロップ条件**: アタッカー（ライバル）を撃破したとき30%の確率でその場に落下する。重力に従って落下し、地面で静止する。
- **ピックアップ**: アタッカーが触れると自動的に所持品に加わる。
- **HUD表示**: HUDのCARRIER表示の直下に、所持数分の小さな緑アイコン（7×7px・白十字・グロー付き）が横並びで表示される。
- **使用タイミング**: キャリアにドッキング（Sキー）したとき、所持しているキット数だけ自動的に消費される。
  - キャリアのHPが満タン未満の場合: **HP 50回復**
  - キャリアのHPが満タンの場合: **キャリア残機+1**
- キットは1個ずつ順番に処理され、複数所持している場合は全て消費される。

#### オートエイムユニット

- **外見**: 16×16px の角丸四角形。オレンジ色（明度がパルスして光り輝くグロー付き）。中央に白いターゲットサークル（二重リング＋十字線）。
- **ドロップ条件**: アタッカー（砲撃型）を撃破したとき30%の確率でその場に落下する。重力に従って落下し、地面で静止する。
- **ピックアップ**: アタッカーが触れると60秒間（3600フレーム）有効になる。**複数取得すると時間が加算され、最大3分（10800フレーム）まで延長できる。**
- **効果（有効中）**:
  - マウスカーソルのワールド座標から半径120px以内に生きている敵がいる場合、照準（クロスヘア）が自動的にその敵の中心へスナップされる（磁石のように吸い付く）。
  - **スティッキーロック**: 一度敵にスナップすると、その敵に照準をロックし続ける。ロック中の敵が撃破されると自動的に次の敵を再スナップする。
  - 有効中は照準が常に**赤色**で表示され、右上に極小の「AUTO」ラベルが付く。スナップ時はL字ブラケット形状に変化する。
  - ミサイル・マシンガン・グレネードの射角、およびアタッカーの向きも自動スナップ先に追従する。
- **スナップ抑制**: 有効中にマウスを1フレームで4px超移動させると照準スナップが一時停止する（タイマーは継続）。マウスを静止させると自動的に再スナップが再開される。
- **HUD表示**: HUDのMISSILE/M-GUN表示の直下に「A-AIM」ラベルと残り時間を示すオレンジの細バーグラフが表示される。バーの全幅は最大時間（3分）を基準とする。時間切れになると消える。

### 3.8 敵メインベース（最終目標）

- マップの最深部（主に右端エリア）に設置された巨大な敵の中枢拠点（アタッカーの約2倍の大きさ）。
- 中央に光り輝くコアを持ち、その外側を3重の防御壁（シールド）が覆っている構造。
- **防衛システム（ベースレーザー）**:
  - プレイヤーまたはキャリアが一定距離（約400px）まで近づくと、コアの中心にエネルギーを充填するモーション（光の粒子の収束）を開始します。
  - 約1.5秒（90フレーム）のチャージ後、ターゲットに向けてエメラルドグリーンの高速レーザーを放ちます。
  - レーザーは地形（ブロック）を無視して直進し、命中すると大ダメージ（HP50%相当）を与えます。
  - 発射後は一定時間のクールダウンが必要です。
- **戦略兵器（長距離巡航ミサイル）**:
  - 7面（Mission 7）以降、ボス基地は遠方で待機しているキャリアを目標として定期的に（20〜30秒間隔）長距離巡航ミサイルを発射します（テスト用として0面から有効）。
  - **発射ワーニング**: 発射の3秒前（180フレーム前）から画面中央に赤いカウントダウンによるワーニングが表示され、ボスの位置から目標までの予測ルートが赤い点線で表示されます。
  - **A*パスファインディング**: ミサイルは単純な追尾ではなく、**A*アルゴリズム**を用いて動的な洞窟地形を解析し、目標までの最短ルートを自律的に探索します。
  - **パススムージング（String Pulling）**: 探索されたグリッドベースのルートは「16×16ピクセルの厚みを持つレイキャスト」を用いたスムージング処理により、壁の角をかすめることなく最短距離で結ぶ滑らかな直線航路へと最適化されます。
  - **射出挙動**: 発射時はまず基地のハッチから斜め上方向（-0.6π）に物理的に射出され、その後スムージングされたルートへと滑らかに合流します。
  - **航行AI**: 巡航中はA*ルートを最優先し、壁との衝突を物理的に回避します。目標にある程度接近すると、最終攻撃フェーズ（ターミナルホーミング）へ移行し、直接ターゲットを追い込みます。
  - **当たり判定**: ミサイルの四隅を含むバウンディングボックスが1ピクセルでもブロックに触れると即座に爆発する「ブロック単位」の精密な判定を採用。命中時は**中ダメージ(30)**と**特大爆発（サイズ40）**を伴う。
  - **迎撃**: ミサイル自体に耐久力があり、プレイヤーのマシンガン3発、またはミサイル/グレネード1発で迎撃・破壊可能です。破壊時も**特大爆発**を伴い、シアン色の軌跡が消滅します。
- ミサイルやグレネードによるダメージは「一番外側の防御壁」から1枚ずつ剥がすように順番に減少していく。
- 3枚の防御壁をすべて破壊した後にコア本体へ攻撃を当てる（計4回のダメージ判定）ことでベースの破壊シーケンスが開始される。
- **破壊シーケンス**: 破壊されると即座に消滅するのではなく、約1.5秒間、基地の各所で**連鎖的な中・大爆発**が発生し、同時に激しい**画面振動（Screen Shake）**を伴う派手な演出が行われる。最後に最大級の爆発とともに消滅する。
- メインベースを破壊すると、その場所に**クリアフラッグ**が出現します。
- 出現したフラッグをアタッカーで奪取することで「MISSION COMPLETE」となりステージクリア。
- スコアとクリア数を引き継いだまま、Wキーまたはクリックで次の自動生成ミッションへ進行する。

## 4. 操作仕様 (PC向けキーボード・マウス操作)

本作はオリジナルの「3方向射撃（上・直進・下）」を発展させ、マウスによる直感的な360度エイミングシステムを採用した操作系とする。

### 4.1 キーボード操作（移動・アクション）

| キー | 通常時 | ドッキング時 |
|---|---|---|
| **A / ←** | アタッカーを左に移動 | キャリアーごと左に移動 |
| **D / →** | アタッカーを右に移動 | キャリアーごと右に移動 |
| **W** | バースト（接地時・燃料80%以上） | ドッキング解除（アタッカー発進）。※頭上に障害物がある場合はキャンセルされる |
| **Shift** | ホバー（空中で長押し、燃料消費） | — |
| **s** | ドッキング開始※ / 長押しでホバー燃料の回復速度UP（裏技） | — |
| **Space** | ミサイルまたはマシンガン発射（現在の武装） | — |
| **F** | ミサイルとマシンガンの切り替え（マシンガンへの切替時にリロード開始）<br>※ミサイル残弾が0になると自動的にマシンガンへ切り替わります。 | — |
| **M** | — (以前のマップキー) | — |
| **R** | 洞窟全体のオーバーレイ・ミニマップ表示（トグル式）。移動開始で自動クローズ | 同左 |

> ※ドッキング条件: アタッカーがキャリアーのプラットフォーム中央付近に接地していること。

#### 移動の物理挙動

- **地上歩行**: 地上ではドリフト（滑り）なし。キーを離すと即座に停止する。
- **空中制御**: 空中ではわずかな慣性（エアフリクション）が働く。
- **速さ制限**: 落下速度（+8.0）に上限がある。また、ホバー中の上昇速度（-6.0）にも上限リミッターがかかる。
- **重力**: 空中では常に下方向の重力が適用される。
- **バースト**: 燃料が80%以上ある場合のみ、接地中にWキーで大量の燃料を消費し瞬時に爆発的な加速（初速度）を得て跳躍する。使用後約0.5秒間はホバーが使用不可になる。
- **ホバー**: バースト後、約0.5秒経過してからWキーを長押しし続けるとホバーが起動し燃料を消費しながら上昇する。
  - **動的推力**: ホバーの推力は燃料残量に応じて変化する。満タン時は重力より強く（上昇可能）、残量が減ると次第に推力が低下し、最低-0.15まで弱まる。
- **しゃがみとスタン**: 一定以上の高さから落下して着地すると「ハードランディング」となり、一定時間しゃがみ状態（スタン）になる。しゃがみ・ドッキング中はバズーカ（ミサイル・グレネード）の発射ができない。
- **離脱時安全確認**: キャリアーから離れる（発進する）際、アタッカーが立ち上がるための頭上スペースにしきい値以上の障害物（ブロック）がある場合は、危険と判断して離脱アクションがキャンセルされる。

### 4.2 マウス操作（攻撃・向き）

| 操作 | 機能 |
|---|---|
| **カーソル移動** | 照準（クロスヘア）の操作 |
| **左クリック / Space** | 照準方向にミサイル発射 |
| **右クリック** | 照準方向にグレネード発射（放物線） |
| **向き自動制御** | 照準がアタッカーの左→左向き、右→右向きに自動切替 |

### 4.3 照準（クロスヘア）の仕様

- 通常時は半透明の十字線で描画。
- **オートエイム有効中**: 照準が赤色になり、右上に極小の「AUTO」ラベルが表示される。
- **ロックオン時**（Shiftキー）: ターゲットを四角く囲む「四隅のL字マーク」へと形状が変化し黄色になる。ロック中は内部マウス座標がクロスヘアの位置に固定されるため、グレネード距離計算やオートエイムのスナップ抑制判定もクロスヘア基準で動作する。
- **クランプ（画面端制限）**: HUDなどのUI領域に描画されないよう上下・左右の全4方向でクランプされる。クランプされた方向には小三角形インジケーター（▲▼◀▶）が表示され、ターゲットが画面外にあることを示す。三角形の頂点はクロスヘアの中心位置に合わせてある。
- 常にマウスカーソルの位置またはロック対象に追従する。
- **ロック解除**: クロスヘアがロック状態であっても、マウスを動かすことで即座にロックが解除され、自由なエイミングに戻る。

## 5. 武器・攻撃システム

### 5.1 ミサイル（メインウェポン）

| 項目 | 仕様 |
|---|---|
| 発射方法 | 左クリック |
| 軌道 | アタッカー位置から照準方向への**直線弾** |
| 速度 | 高速 |
| 威力 | 通常ブロック1発破壊。敵ユニットに中ダメージ |
| 弾数 | 有限（初期24発）。キャリアードッキングで全回復 |
| 同時発射数 | 画面内に最大10発まで同時存在可 |

#### ミサイル (Missile)

- 直進するミサイル。高威力(15)で地形を破壊可能。
- 初期弾数24、同時発射数制限あり。

#### マシンガン砲 (Machine Gun)

- **手動切替**: **Fキー**を押すことで、ミサイルとマシンガンを任意に切り替えることができる。
- **リロード開始**: ミサイルからマシンガンに切り替えた瞬間、マシンガンのリロードが自動的に開始される（即時リロードではなく、一定のリロード時間が必要）。
- **性能**:
  - 16発のバースト射撃を行い、全弾発射後に約1秒間のリロード時間が発生する。弾数は無限。
  - 威力はミサイルの1/5 (3)。
  - **射程**: ミサイルのちょうど半分 (720px)。
  - **弾速**: 敵の弾と同じ (3)。寿命は240フレーム。
  - **精度**: 狙いが多少ばらつき、クロスヘア方向にランダムな散らばりが発生する。
  - 地形（ブロック）や**敵メインベース（ボス）**を破壊・ダメージを与えることはできない。
  - 発射音や弾丸の軌道はタレットの攻撃に類似する（黄色い小弾丸）。

#### グレネード (Grenade)

- 放物線を描いて飛び、着弾または一定時間で爆発する。爆風は広範囲の地形と敵に大ダメージ(80)を与える。
- 初期弾数12。自機への被弾ダメージはない。キャリアも巻き込まれるため注意が必要。

| 項目 | 仕様 |
|---|---|
| 発射方法 | 右クリック |
| 軌道 | 照準方向への**放物線（アーク）弾**。投下後、地面や壁を跳ね返りながら転がる。落下速度の上限あり。 |
| **投擲強度** | プレイヤーとマウスカーソルのワールド距離に比例（最小: 0、最大: 8、距離200px以上でMAX）。近くにカーソルを置くとゆっくり手前に落とし、遠くに置くと力強く投擲する。 |
| 弾数 | 有限（初期12発）。キャリアドッキングで全回復 |
| 爆発範囲 | 着弾点を中心に周囲ブロックも巻き込み破壊。約2秒後に自動爆発。 |

### 5.3 キャリアー武装（自動防衛）

- 短射程の自動ミサイルを前方に発射（弾数無制限）。
- 射程は限定的で、接近する敵への自衛用。

### 5.4 爆発・ダメージエフェクト

- 着弾・破壊時に黄色〜白のピクセルクラスタが一瞬フリッカーするレトロ風爆発エフェクト。
- ブロック破壊時は破片パーティクル（小さな四角形）が飛散。
- 自機や敵機がダメージを受けた際、上方向にオレンジ色の火花（Sparks）が散る演出が発生。

## 6. 主なゲームメカニクス

### 6.1 ドッキングと補給システム

ゲームの根幹をなす最重要メカニクス。

```
【ドッキングシーケンス】

  アタッカーがキャリアー上部に接地
          ↓
     [S]キーでドッキング開始
          ↓
  ドッキング完了アニメーション
          ↓
  ミサイル / グレネード / ホバー燃料 / HP 全回復
          ↓
  【キャリアリペアキット処理】（所持数分ループ）
    ├── キャリアHP < 満タン → HP+50
    └── キャリアHP = 満タン → キャリア残機+1
          ↓
  [A][D]でキャリアーごと移動可（前線構築）
  ※この状態では射撃不可（しゃがみポーズ）
          ↓
  [W]キーでアタッカー発進（頭上クリア確認後）
```

### ドッキングとキャリア操作（W/A/S/Dキー）

プレイヤーは「キャリア」と呼ばれる補給母艦から出撃し、作戦を行います。

- **ドッキング（Sキー）**: キャリアの上に乗り、Sキーを押すことでドッキング状態になります。
  - ドッキング中は自動的にHP、ミサイル、手榴弾、ホバー燃料が満タンに回復します。
  - キャリアに乗ってA・Dキーで左右に安全に移動・横断が可能です。
- **アンドック（Wキー）**: ドッキング中にWキーを押すと、上空に飛び降りて単独行動を開始します。
- **キャリア防衛と敵の標的**:
  - 敵（戦車・アタッカー等）は距離に応じてプレイヤーまたはキャリアを標的として狙ってきます。
  - プレイヤーがドッキングしている間は、敵の攻撃はキャリアに集中しやすくなります。
- **キャリア破壊・強制離脱とリスポーン**:
  - 敵の攻撃によりキャリアのHPが0になるとキャリアは爆発し、残機（LIVES）が1減少します。
  - ドッキング中にキャリアが破壊された場合、プレイヤーは自動的に上空へ放り出され（強制離脱）、単独操作に戻ります。
  - キャリアは残機がある限り、初期のスタート地点付近にリスポーン（復活）します。キャリアの残機が0になった状態で破壊されると、プレイヤーの残機に関わらずゲームオーバーになります。掃討 → 安全確認 → キャリアーを前進 → 再ドッキングして補給、を繰り返す。

### 6.2 物理挙動とエンティティの相互干渉

- **重力**: アタッカーには常に下方向の重力が適用される。
- **ホバー**: 燃料を消費して重力に逆らい、上昇・滞空が可能。
- **慣性**: 移動停止後も少しだけ滑空する（空気抵抗/摩擦のシミュレーション）。
- **キャリア押し上げ**: アタッカーが上昇中（vy < 0）にキャリア底面に接触すると、キャリアを下から担いで押し上げることができる。ただしキャリアは重いため、アタッカーの上昇速度の半分（×0.5）でしか持ち上がらない。また、押し上げ中はキャリアがアタッカーの左右移動に追従するが、横方向へ直接押すことはできない。
- **エンティティの相互干渉（当たり判定）**:
  - プレイヤーと敵キャラクター（アタッカー、戦車）はお互いにすり抜けない「剛体（ソリッド）」として干渉します。
  - 横からぶつかった場合は「壁」として機能し、押し返されます。
  - 敵の頭上にジャンプ・落下した場合は「足場」として着地することができ、敵の上に乗ったまま移動することが可能です。
  - 敵同士も同様に干渉し、例えばパトロール中の死角で敵同士が重なることを防ぎます（ぶつかると方向転換します）。さらに生成時（スポーン時）も重ならないようばらけて配置されます。
  - **フレンドリーファイア無効**: 敵味方の識別があり、敵が放った弾や爆風は他の敵ユニットにはダメージを与えません。
  - 戦車（EnemyTank）は1段差の壁であればスムーズに登り、1段差の崖であれば降りることができます（2段以上の崖は反転して引き返します）。
- **ホバー燃料の自動回復**: ホバーを使用していない間、燃料は自動的に回復する（回復速度: 0.5/frame、満タンまで約3.3秒）。キャリアーとのドッキング時は即座に全回復。
  - **🕹️ 裏技**: キャリアーから離れた状態で`S`キーを長押しすると、ホバー燃料の回復速度が大幅に上昇する（0.75/frame）。通常時(0.5)の1.5倍の速さでチャージ可能。キャリアー付近では通常のドッキング操作が優先される。
- **慣性**: 地上移動時、入力停止後も僅かに滑る（氷上ほどではない微慣性）。
- **しゃがみジャンプ**（オリジナル準拠）: 特定操作で通常より高いジャンプが可能（実装検討）。

### 6.3 破壊可能地形

洞窟を構成するブロックの多くは攻撃で破壊可能。

- **戦略的用途**:
  - ルート開拓: 壁を壊して新たな通路を作る。
  - 射線確保: 安全な位置から壁を壊し、敵を狙い撃つ。
  - 落とし穴: 足元のブロックを壊して下層へ降りる。
  - キャリアー通路確保: キャリアーが通れる幅の通路を確保する。
- **破壊演出**: ブロック破壊時は小さな破片パーティクルが散ってから消滅。

### 6.4 マウスエイムによる360度攻撃

- マウスカーソル位置 = 照準位置。アタッカーの位置から照準方向にミサイル/グレネードを発射。
- **移動と攻撃の独立制御**: 左に移動しながら右にミサイルを発射する「引き撃ち」、上昇しながら下方にグレネードを投げるなどの高度な戦術が可能。
- アタッカーの向き（左向き/右向き）は照準位置により自動制御される。

### 6.5 リソース管理

本ゲームはリソース管理がゲーム性の核。

| リソース | 消費条件 | 回復条件 | 枯渇時 |
|---|---|---|---|
| ミサイル | 左クリックで1発消費 | キャリアーとドッキング | 発射不可 |
| グレネード | 右クリックで1発消費 | キャリアーとドッキング | 発射不可 |
| ホバー燃料 | Shift長押しで継続消費 | **自動回復**（未使用時0.5/f）/ ドッキングで全回復 / S長押しで+20%(裏技) | ホバー不可、落下 |
| アタッカーHP | 被弾・トラップ接触で減少 | キャリアーとドッキング（※検討） | アタッカー撃破 |

### 6.6 スクロールとカメラ

- **追従カメラ**: アタッカー（操作対象キャラ）を中心にスムーズに追従。
- **補間**: 急なカメラジャンプを防ぐ線形補間（lerp）によるスムーズパン。
- **マップ境界クランプ**: カメラはマップ外には出ない。

### 7.2 初期状態

- アタッカー残機: 3機
- キャリアー残機: 3機
- ドッキング状態でステージ左上付近からスタート。
- ミサイル・グレネード・ホバー燃料は満タン。

### 7.3 撃破・リスポーン

- **アタッカー撃破時**: 爆発エフェクト → 残機-1 → キャリアー上に新アタッカーが出現（ドッキング状態）。
  - **再出撃の即時無敵**: リスポーンした瞬間から一定時間（約2秒間）、**無敵状態となり機体が点滅**する。
  - **状態の完全リセット**: リスポーン時にはそれまでのホバリング状態、しゃがみ、スタン、および動作音（ホバー音等）がすべてクリアされ、クリーンな状態で再開される。
- **キャリアー撃破時**: 大爆発エフェクト → 残機-1 → （ドッキング中であれば同時にアタッカーも失う）。
- **ゲームオーバー条件**: キャリアーの残機が0の状態でキャリアーが撃破される。

| 行動 | スコア |

| 通常敵撃破 | 100〜500点（種類別） |
| 固定砲台撃破 | 300点 |
| 飛行ユニット撃破 | 500点 |
| ミッション目標破壊 | 1000点 |
| 敵メインベース破壊 | 5000点 |
| タイムボーナス | フラッグ奪取時（ミッションクリア時）に加算されるボーナス。**マップの面積（総タイル数）に比例**して設定される（ベースボーナス ≒ 総タイル数）。1秒経過ごとに50点減算（下限0点）。ゲームループを停止させ、画面左下のクリア画面にてカウントアップ演出で加算される。 |

### 7.5 全ステージクリア（ゲームクリア）

- 本ゲームは全7ステージ（ミッション1〜7）構成とする。
- 7面のメインベースを破壊しフラッグを奪取するとゲームクリアとなり、「CONGRATULATIONS!」の画面が表示されてゲーム終了となる。

### 7.6 タイトル画面・ランキング入力の画面遷移

- **タイトル画面（静寂モード）**
  - タイトル画面ではBGMが終了し、静寂な状態となる。
  - タイトル画面のまま一定時間経過すると、ハイスコアトップ20の表示画面に切り替わる。点滅テキストは「Press Any Key to Start」とする。
  - そのまま一定時間経過すると、再びタイトル画面へとループする。
  - このループ（タイトル ↔ トップ20表示）の間にキーが押されるとゲームスタート（1面開始）となる。
- **ランキング入力画面（明るいBGM）**
  - ゲームオーバー時、または7面ゲームクリア時点でのスコアが「ハイスコアトップ20」に入っている場合、ランキング入力画面へ進む。
  - ランキング入力画面に進むと、明るいムードの音楽にBGMが切り替わる。
  - 左寄せで入力が完了すると、トップ20のリストが表示され、今回新しく追加された自分の行がマゼンタ色でハイライト点滅する。
  - ランキング表示終了後、タイトル画面に戻ると同時にBGMも**タイトルの曲（title.mp3等）へスムーズに切り替わる**。BGM停止時は約0.5秒のフェードアウトを伴う。

## 8. マップ・レベルデザイン

### 8.1 マップ仕様

- **タイル（グリッド）ベース**: 固定サイズのタイルで構成（例: 32×32px/タイル）。
- **マップ全体サイズ**: 横100タイル × 縦40タイル程度（3200×1280px）のスクロールマップ。
- **進行方向**: 全体として左上（スタート）→ 右下（ゴール）へ進む構成。
- **レイヤー構造**: 背景レイヤー（暗黒の洞窟背景）+ タイルレイヤー（ブロック）+ エンティティレイヤー（キャラクター、敵、トラップ）。

### 8.2 マップ生成

- **Rooms and Tunnels方式（アリの巣状構造・大空間特化）**を使用した手続き型生成アルゴリズムによるハイブリッド方式。
- ステージ構成手順:
  1. スタート（左上）、ゴール（右下）、敵配置用の巨大な大部屋を複数確定で配置。
  2. さらにランダムな位置に、非常に巨大な部屋（チャンバー）を35〜49個程度大量に生成。
  3. すべての部屋が孤立しないよう、太いトンネル（幅6〜11マス）で結びつけ、マップ全体の多くを「空洞」にする。
  4. セルオートマトンによるスムージングを数パス行い、直線的な壁を有機的で自然な洞窟の形状（アリの巣風）に削り出す。
  5. 空間が広すぎて足場がない場所に対して、ランダムに横長の**浮遊ブロック（プラットフォーム）**を多数配置する。
- その後、破壊可能なハードブロック、トラップ、敵ユニットを空洞やプラットフォーム上に配置する。
- 全体として、ステージには以下のエリア構成要素を含む:
  - **スタートエリア**: 左上に配置された超巨大な安全地帯。キャリアーの初期位置。
  - **中間通路/大空洞**: 太い通路と巨大な部屋が連なり、足場（プラットフォーム）が浮かぶ広大なメインエリア。
  - **敵配置エリア**: 敵ユニットとトラップが密集するランダム配置の巨大な危険地帯。
  - **ボスエリア**: 敵メインベースが設置された右下最深部の超巨大な空間。

### 8.3 全体マップ表示

- **R**キーで画面中央に広域マップを半透明のオーバーレイ表示。一度生成したオフスクリーンキャンバスを用いて軽量に描画。
- プレイヤーが移動（A/D/Wキー）を開始すると自動的にフェードアウトしながら閉じる。
- 地形の状態に加え、現在のアタッカー位置（白）、キャリアー位置（青）、生きている敵ユニットの位置（赤）がリアルタイムに表示される。

## 9. 技術仕様

### 9.1 描画

- HTML5 Canvas 2Dコンテキストによるレンダリング。
- `requestAnimationFrame`によるゲームループ（可変フレームレート、deltaTime制御）。
- 描画最適化: カメラ範囲内（ビューポート内）のタイルとエンティティのみ描画。

### 9.2 ゲームループ

```
requestAnimationFrame(loop)
  └── loop(timestamp)
       ├── deltaTime = timestamp - lastTime
       ├── update(deltaTime)
       │    ├── Input処理
       │    ├── 物理演算（重力、慣性、衝突判定）
       │    ├── エンティティ更新（プレイヤー、敵、弾）
       │    └── カメラ更新
       └── draw()
            ├── 背景描画
            ├── マップ描画（ビューポート範囲のみ）
            ├── エンティティ描画
            └── HUD描画（カメラ座標とは独立）
```

### 9.3 衝突判定

- タイルベースの矩形衝突判定（AABB）。
- エンティティ同士の衝突判定（弾 vs 敵、弾 vs ブロック、プレイヤー vs トラップ etc.）。

### 9.4 モジュール構成

## 10. サウンド・SE（実装済み）

ゲーム内のサウンドはすべてWeb Audio APIを使用したプロシージャル生成で実装されています。

| イベント | SE概要 |
|---|---|
| ミサイル発射 | 短い破裂音「パシュッ」 |
| グレネード発射 | 重めの発射音「ドン」 |
| 着弾・爆発 | 爆破音「ボン」 |
| ブロック破壊 | ガラガラ崩れる音 |
| ホバー中 | 継続的なジェット噴射音「シュー…」 |
| ドッキング | メカニカルな接続音「ガシャン」 |
| 被ダメージ（自機・キャリア） | 金属が激しくぶつかり歪む、重厚で鋭い低音（ディストーション/ノイズ混じり）。敵へのダメージ音（火花）とは明確に異質な音色。 |
| 敵撃破 | 軽快な爆発音（自機ダメージ音より高く、クリアな響き） |
| ゲームオーバー | プレイヤー残機ゼロおよびキャリア撃破時に流れる。重く悲しい下行アルペジオによるファンファーレ（ローパスフィルターで徐々にこもってフェードアウト） |
| 敵基地破壊（フラッグ出現） | 短い上昇ジングル（タ・タ・ターン） |
| ステージクリア（フラッグ奪取） | 凱旋を象徴する長めの荘厳なファンファーレ（メロディ、ハーモニー、ベース、アルペジオが重なる多層サウンド） |
| メインBGM | ミッションごとに対応したMP3ファイル（`bgm01.mp3`〜`bgm07.mp3`）を再生。ゲーム開始時・ミッションクリア後の次ステージ進行時にステージ番号を基に正しいBGMへ切り替わる。 |
| ランキング入力BGM | 明るいムードの音楽。ゲーム終了後にハイスコアトップ20に入ると再生される。入力終了・ランキング表示後タイトル画面に戻ると停止し、静寂に戻る。 |
